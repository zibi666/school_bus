# School Bus Frontend

基于 Vue 3 + Vite + Element Plus 开发的校园包车系统前端页面。

## 🛠 技术栈

- **核心框架**: Vue 3 (Composition API)
- **构建工具**: Vite
- **UI 组件库**: Element Plus
- **路由管理**: Vue Router 4
- **HTTP 客户端**: Axios

## 🚀 快速开始

### 1. 环境准备

- Node.js 16+
- npm 或 yarn

### 2. 安装依赖

在当前目录下运行：

```bash
npm install
```

### 3. 配置环境变量

项目根目录下已创建 `.env` 文件，用于配置后端接口地址：

```properties
VITE_API_BASE_URL=http://localhost:8080/api
```

如果你的后端服务运行在其他地址，请修改此文件。

### 4. 启动开发服务器

```bash
npm run dev
```

启动后访问控制台输出的本地地址（通常是 `http://localhost:5173`）。

## 📱 功能模块

### 学生端
- **注册/登录**: 学号登录，支持填写所在地。
- **包车申请**: 填写目的地、用车时间、车型需求。
- **我的订单**: 查看历史订单状态（审核中/已通过/已拒绝），支持取消审核中的订单。
- **个人中心**: 查看个人基本信息。

### 管理员端
- **登录**: 管理员账号登录。
- **订单审核**: 查看所有申请，进行通过（分配车辆）或拒绝操作。
- **车辆管理**: 录入车辆信息，查看车辆状态（空闲/使用中），删除车辆。

## 🎨 交互体验优化

### 1. 全局加载反馈
- **LoadingSpinner 组件**: 封装了统一的加载动画组件，支持自定义提示文本。
- **全覆盖集成**: 在登录、注册、个人中心、订单列表、车辆管理等所有涉及异步请求的页面集成加载状态。
- **智能遮罩**: 关键操作（如提交表单）采用全屏遮罩，防止重复提交。

### 2. 骨架屏优化 (Skeleton Screens)
- **订单列表**: 首次加载时展示骨架屏，模拟真实数据结构，提升视觉流畅度。
- **发车详情**: 针对复杂的发车详情页定制了 `SkeletonBusTrip` 组件，完美匹配页面布局。
- **管理员后台**: 为订单审核列表定制了 `SkeletonTable` 组件，为车辆管理定制了 `SkeletonCard` 组件，保持统一的加载体验。
- **智能缓存**: 实现了前端内存缓存机制，已有数据时不再重复显示骨架屏，避免闪烁。

### 3. 视觉与动效
- **界面美化**: 优化了发车页面的卡片样式，增强了对比度和层级感。
- **过渡动画**: 添加了平滑的展开/折叠动画和加载状态切换动画。
