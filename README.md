---



# 一、项目简介

本系统是一个面向校园场景的 **学生汽车包车预定系统**，学生可在线提交包车申请，管理员在后台进行审核与车辆分配，实现包车业务的规范化、信息化管理。系统支持完整的 **申请 → 审核 → 派车 → 通过/拒绝 → 查询** 全流程。



---



# 二、网页页面设计

## 1. 登录注册页面

- 学生登录 / 注册
- 管理员登录

## 2. 学生端功能页面

- **包车申请页面**（填写包车申请单）
- **我的订单列表页面**（显示所有状态的订单：审核中 / 已通过 / 已拒绝）
- **订单详情页面**（展示分配的车辆信息或拒绝理由）
- **个人信息页面**

## 3. 管理员端功能页面

- **订单审核页面**（列表显示待审核订单，包含“通过”和“拒绝”按钮）
- **车辆管理页面**（录入和管理车辆资源）

# 三、数据库表设计

## 1. 学生基本信息表（t_student）

- **学号（主键）**
- 姓名  
- 密码  
- 所在地  



---

## 2. 管理员信息表（t_admin）

- **管理员ID（主键）**
- 账号  
- 姓名  
- 密码  



---

## 3. 车辆信息表（t_bus）

- **车辆ID（主键）**
- 车牌号  
- 车型  
- 司机姓名  
- 车辆状态（空闲 / 使用中）  



---

## 4. 学生购票 / 包车订单表（t_order）

- **订单号（主键）**
- **学号**（关联学生表）
- **目的地**
- **使用时间段**
- **申请车型**
- **分配车辆ID**（审核通过后填充，关联车辆表）
- **订单总费用**（审核通过后填充）
- **订单状态**（审核中 / 已通过 / 已拒绝）
- **拒绝理由**
  - 当状态为“已拒绝”时必填  
  - 其他状态可为空  
- 创建时间

# 四、学生端功能说明

## 1. 学生登录

- 学生输入 **学号** 和 **密码** 登录系统。



---

## 2. 申请包车

- 学生填写包车申请单并提交。
- 提交成功后系统自动生成一条 **状态为“审核中”** 的订单。
- 必填项包括：
  - **目的地**
  - **使用时间段**
  - **需求车型**



---

## 3. 查看所有订单

- 学生进入“我的订单”页面，可查看自己提交的 **全部历史订单**。
- 包含三种状态：
  - **审核中**
  - **已通过**
  - **已拒绝**

### 状态展示逻辑说明：

- **审核中**：  
  显示“等待管理员处理”。

- **已通过**：  
  显示“审核通过”，并展示以下信息：
  - 车牌号  
  - 司机姓名  
  - 联系电话  
  - 订单费用  

- **已拒绝**：  
  显示“审核不通过”，并 **醒目展示管理员填写的拒绝理由**  
  （如：车辆不足、目的地无法到达等）。




---

## 4. 终止申请

- 在管理员 **尚未审核之前**，学生可以主动取消自己的包车订单。



---



# 五、管理员端功能说明

## 1. 管理员登录

- 管理员输入 **账号** 和 **密码** 登录后台管理系统。



---

## 2. 审核学生的包车申请

管理员可查看所有 **状态为“审核中”** 的订单，并进行以下两种处理方式：



---

### 操作 A：审核通过（分配车辆）

- 管理员为该订单选择一辆 **状态为空闲的车辆**。
- 输入 **订单总费用**。
- 点击“通过”按钮：
  - 系统自动将订单状态更新为：**已通过**
  - 同时完成车辆分配与价格录入。



---

### 操作 B：审核不通过（拒绝）

- 管理员点击“拒绝”按钮。
- **必须填写拒绝理由**
  - 例如：  
    - 该时间段无空闲大巴  
    - 申请信息填写错误  
    - 目的地超出服务范围等  
- 点击确认后：
  - 系统将订单状态更新为：**已拒绝**
  - 并将拒绝理由写入数据库，供学生查看。



---

## 3. 车辆资源管理

### 新增车辆

- 管理员可录入：
  - 车牌号
  - 车型
  - 司机信息

### 状态维护

- 管理车辆 **空闲 / 使用中** 状态。
- 保证 **同一时间不会将同一辆车分配给多个订单**。
